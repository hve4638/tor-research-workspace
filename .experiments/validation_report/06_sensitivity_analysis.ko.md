# 민감도 분석: Guard 수명 파라미터 스윕

**검증 테스트**: S3 (Guard 수명 민감도)
**데이터 소스**: `data/s3_guard_lifetime_sensitivity.json` (S1-S3 시뮬레이션 완료 대기 중)

## 1. S3: Guard 수명 민감도

### 1.1 방법론

Guard 수명 파라미터(균등 분포 범위)는 클라이언트가 guard 릴레이를 교체하는 빈도에 직접 영향을 미친다. 짧은 수명은 더 빈번한 교체를 의미하며, 이는 시간이 지남에 따라 침해된 guard를 선택할 확률을 증가시킨다. 이 민감도 분석은 guard 수명 범위를 변화시켜 다음에 대한 영향을 정량화한다:

- 클라이언트 침해율 (릴레이 적대자 시나리오)
- 첫 침해까지의 시간
- 누적 상관률 (AS 수준 관찰자)

### 1.2 파라미터 공간

Tor 사양은 uniform[30, 60]일에서 추출한 guard 수명을 사용한다. 민감도 스윕 테스트:

- 짧은 수명: uniform[15, 30]일 (빠른 교체)
- 기본값: uniform[30, 60]일 (기준선)
- 긴 수명: uniform[60, 120]일 (느린 교체)

### 1.3 결과

*다중 시드 시뮬레이션 완료 대기 중. 데이터가 이용 가능해지면 `data/s3_guard_lifetime_sensitivity.json`에서 결과가 채워질 예정.*

예상 발견:
- 짧은 guard 수명은 침해 확률을 증가시켜야 함 (더 많은 교체 = 적대자 guard 선택 기회 증가)
- 관계는 작은 적대자 대역폭 점유율에서 대략 선형이어야 함
- 방어 메커니즘이 차별적 민감도를 보여야 함 (Counter-RAPTOR가 Astoria보다 더 영향받음)
